Privacy Policy

Extension Name: RCRA BI Portal
Effective Date: February 8, 2026

Summary: The RCRA BI Portal extension does not collect, transmit, or sell your personal browsing history. All data processing happens locally on your device.

1. Data Collection

The RCRA BI Portal extension is designed to act as a navigation overlay. It minimizes data collection to the absolute necessities required for functionality:

  • No Tracking: We do not track your browsing history, search queries, or website interactions.
  • No Remote Server: The extension does not send data to any third-party server owned or operated by the developer.
  • Local Preferences: Settings such as sidebar position (left/right), display mode (mini/full), and cached menu data are stored locally on your browser using Chrome's storage.local API.

2. Permissions Usage

The extension requires specific browser permissions to function. Here is an explanation of why each permission is necessary:

  • "Read and change all your data on the websites you visit" (<all_urls>):
    This permission is strictly used to inject the visual sidebar (iframe) into the websites you visit. The extension needs to modify the page layout (CSS margins) to ensure the sidebar does not cover up website content. It does not read, store, or analyze the content of the pages you visit.
  • Storage:
    Used to save your user preferences (e.g., keeping the sidebar open or closed) and to cache the navigation menu for faster loading times.
  • Scripting:
    Required to execute the code that renders the sidebar overlay.

3. External Data Connections

The extension makes read-only requests to GitHub to fetch the latest navigation menu configuration (menu2.json). This ensures your navigation links are always up to date without requiring an extension update.

4. Admin Features (Optional)

For administrators using the "Menu Editor" feature:

  • If you choose to save GitHub credentials (Personal Access Token), this sensitive information is saved locally in your browser's local storage.
  • It is never transmitted to the developer or any tracking service.
  • It is used solely to authenticate direct requests between your browser and the GitHub API for saving menu changes.
